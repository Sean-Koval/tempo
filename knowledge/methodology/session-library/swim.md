# Session Library — Swim

Canonical swim sessions used in `plan.yaml` and `sessions_planned.library_ref`. Each entry has a stable `id` (snake_case), target duration/TSS range, purpose, and execution structure.

## Swim

### `technique_swim`
- **Purpose:** stroke quality, not fitness
- **Duration:** 45–60 min
- **TSS:** ~30–45
- **Structure:** 400 WU → 4×50 drill focus (catch-up, fingertip-drag, sculling) → 8–12×50 @ stroke-count-target → 400 CD

### `aerobic_swim_set`
- **Purpose:** Z2 swim volume
- **Duration:** 45–75 min
- **TSS:** ~45–65
- **Structure:** 400 WU → 3–5×600 @ CSS+10" with 30" rest → 200 CD

### `race_pace_swim`
- **Purpose:** open-water race-pace rehearsal
- **Duration:** 60–75 min
- **TSS:** ~55–75
- **Structure:** 400 WU → 6–10×200 @ race effort with sighting → 200 CD

### `sprint_50s_race_pace`
- **Purpose:** neuromuscular speed + tolerance for race-start surges
- **Duration:** 45–60 min
- **TSS:** ~40–60
- **Structure:** 400 WU → 4×50 build → main: 8–12×50 @ goal-race-pace or faster, on 1:30 (or 20–30" rest) → 200 EZ → optional 4×25 fast → 200 CD
- **Notes:** target stroke-rate consistent across reps; dropping more than 1s off pace = end the set. Use for masters meets and to seed front-of-pack swim starts.

### `css_ladder`
- **Purpose:** raise critical swim speed (CSS) — the IM-relevant aerobic ceiling
- **Duration:** 60–75 min
- **TSS:** ~55–75
- **Structure:** 400 WU → main pyramid 100/200/300/400/300/200/100 @ CSS pace, 15" rest → 200 CD. Alternates: 6×100 + 4×200 + 2×300 ladder; or 5×400 @ CSS+0–2"/100. Anchor pace from a recent 400+200 CSS test.
- **Notes:** CSS pace is sustainable for ~30 min; if last rep falls > 3s/100 off the first, end the set early (R-1 spirit).
