# Ironman build structure — Tempo synthesis

The specific shape Tempo uses when bootstrap-plan builds a 24-week full-IM plan or 16-week 70.3 plan. Drawn from Friel, CTS, Couzens, and Dixon with an aerobic-first, durability-focused bias.

## Principles

1. **Aerobic capacity first, intensity later.** Base is long and patient. Rushing into race-pace work before the aerobic engine is developed wastes fitness.
2. **Durability over peak TSS.** The IM is won by the athlete who's still running at km 35, not the one who set the highest CTL.
3. **Race-pace precision beats threshold chasing.** In Build/Peak, the highest-value bike session is at race NP for race-duration, fueled correctly. Threshold work is secondary.
4. **Fueling is a trained ability.** Every long session past 3h rehearses the race plan. Gut capacity is as trainable as aerobic capacity.
5. **Swim is a force multiplier, not the winner.** Swim well enough to exit fresh; bike and run are where races are decided.

## Weekly shape (typical Base/Build peak week)

| Day | Session | Notes |
| --- | --- | --- |
| Mon | Swim (technique) + easy 30min bike OR rest | Ease into the week |
| Tue | Bike threshold OR run cruise intervals | Hardest intensity of the week |
| Wed | Swim (aerobic) + run easy | Recovery-oriented |
| Thu | Bike race-pace OR tempo block | Second intensity, complementary to Tue |
| Fri | Swim (optional) + rest or easy Z1 | Pre-long-ride recovery |
| Sat | Long ride (Z2, phase-dependent length) | Anchor session |
| Sun | Long run (Z2) OR brick off Sat ride | Second anchor |

## TSS targets by phase (for ~10h/wk athlete)

| Phase | Weekly TSS | Peak week | Peak long ride | Peak long run |
| --- | --- | --- | --- | --- |
| Anatomical | 350–450 | 500 | 2h | 60min |
| Base | 500–700 | 750 | 4h | 100min |
| Build | 650–850 | 900 | 5h | 130min |
| Peak | 650–800 | 850 | 5h + 60min brick run | race sim |
| Taper | 350–500 | — | 2h openers | 45min primer |

Scale proportionally to athlete's weekly hour budget from `preferences.md`.

## Key sessions to land in each phase

- **Base:** 2–3× `long_ride_z2` reaching 80% of race bike duration. 2× `long_run_z2` reaching 60% of expected race run duration.
- **Build:** 2–3× `race_pace_bike` at race NP for ≥50% of race bike duration. 1–2× `brick_run_off_bike` with race-pace fueling.
- **Peak:** 1–2× `race_sim_brick` at full race pacing and fueling. 1× tune-up race (70.3 or long bike TT).
- **Taper:** no new stimulus. Short, fast, fueled openers.

## Common failure modes to guard against

- **Skipping recovery weeks.** Handled by R-15 in decision-rules.
- **Long-run explosion.** Handled by R-14.
- **First-time fueling on race day.** Handled by R-17/R-18.
- **Pyramid drift.** Too much Z3 creeping into "easy" days. Reviewed in weekly post-mortem.
- **Over-testing.** FTP tests and tune-up races that siphon adaptation energy. Bootstrap-plan schedules at most 2 field tests per macrocycle.
